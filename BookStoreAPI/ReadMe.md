# DemoQA BookStore API Automated Testing

This project is a Python-based automated testing suite for the [DemoQA BookStore API](https://demoqa.com/swagger/). It utilizes `pytest` for test orchestration and a custom API client wrapper to interact with the service endpoints.

## Project Structure

```text
BookStoreAPI/
├── tests/
│   ├── test_account_api.py     # Tests for User creation, Token generation, and Authorization
│   └── test_bookstore_api.py   # Tests for Book catalog and User collection management
├── utils/
│   ├── api_client.py           # Custom wrapper for HTTP requests
│   └── helper.py               # Utilities for environment variable management
├── conftest.py                 # Pytest fixtures and global setup/teardown
├── .env                        # Environment configuration (Base URL, Tokens, IDs)
└── requirements.txt            # Project dependencies
```

## Features

- **Account Management**: Tests for creating users, generating authentication tokens, and verifying authorization status.
- **BookStore Operations**: Tests for retrieving the book list, getting specific book details, and managing a user's book collection (Add, Update, Delete).
- **State Persistence**: Uses a `.env` file and helper utilities to share dynamic data (like `USER_ID` and `API_TOKEN`) across different test functions.
- **Automatic Teardown**: A session-scoped fixture in `conftest.py` ensures that the test user created during the run is deleted from the system after all tests are completed.

## Prerequisites

- Python 3.8+
- Pip (Python package manager)

## Installation

1. **Clone the repository**:
   ```bash
   git clone <repository-url>
   cd BookStoreAPI
   ```

2. **Install dependencies**:
   ```bash
   pip install pytest python-dotenv requests
   ```

3. **Configure Environment**:
   Create a `.env` file in the root directory with the following content:
   ```env
   BASE_URL=https://demoqa.com
   ```

## Running the Tests

To execute the entire test suite, run:

```bash
pytest -v -s
```

### Test Workflow
1. **Account Setup**: `test_account_api.py` creates a user and stores the credentials and token.
2. **Book Operations**: `test_bookstore_api.py` uses the generated token to interact with the bookstore collection.
3. **Teardown**: The `test_delete_user` fixture in `conftest.py` automatically cleans up the created user via the API.

## Key Components

- **`api_client` fixture**: Provides a pre-configured session with base URLs and authorization headers.
- **`update_env_variable`**: A helper function that updates the `.env` file in real-time, allowing subsequent tests to access data generated by previous tests (like the `USER_ID`).
